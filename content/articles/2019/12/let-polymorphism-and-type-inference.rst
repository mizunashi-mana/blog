let 多相と型推論
================

:date: 2019-12-20 20:00
:tags: Haskell, 多相関数, 型システム, 型推論
:category: プログラミング言語

Haskell をはじめとする，ラムダ計算を基本的なモデルにし多相的な型を持つ多くの言語では， let 多相 (let polymorphism) に関する理論を基礎としている．この理論にも，色々亜種はあるのだが，ここではとりあえず Haskell に関係する基礎的な理論の話を紹介したいと思う．動機として，こういう話を結構人に説明する場面が多い気がしたので，その練習をしたいという感じ．なので，内容は形式的というより，むしろ理論的な話を抜きにして実装と感覚よりの話になってくると思う．ここら辺の理解の１番の近道は， TaPL (Types and Programming Languages) などのちゃんとした文献読んで，ちゃんと自分で実装してみることだと思うので，勉強したい系の人にはそちらを強くお勧めする．この記事はどっちかというと，てっとり早く知りたい人向け．ただ，この文を書いてる人は割と初心者なので，間違いが結構あるかも．そこは注意．

最終的に目指すのは， `Haskell Language Report 2010 - 4.5 Static Semantics of Function and Pattern Bindings <https://www.haskell.org/onlinereport/haskell2010/haskellch4.html#x10-880004.5>`_ の言ってることが読者になんとかなく理解できるようになることだ．

単相型しかない世界
------------------

さて， Haskell を触ってる人たちにとって多相型がある世界は割と当たり前の世界だと思うけど，その世界は実はかなり難しいことで満ちている．なので，みなさんはそんな難しい世界に身をおけて一つでもプログラムが書けてる時点でかなり素養があると思う．ただ，いきなりそんな世界の話を取り扱ってもあまりよく分からないので，ここではまず多相型を考えない世界で，幾つかの問題を考えてみたいと思う．軽いジョブだと思って付き合ってくれ．
